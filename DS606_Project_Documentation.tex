\section{Executive Summary}

This project investigates whether safety guardrails learned in English transfer to other languages (Hindi, Bengali, Hinglish) in Large Language Models. We are training and evaluating Meta-Llama-3-8B with alignment techniques (SFT + DPO), then testing robustness using cross-lingual jailbreak prompts and implementing refusal direction analysis.

\textbf{Key Findings Target:}
\begin{itemize}
    \item Measure safety alignment transfer from English to Hindi
    \item Identify which harm categories show largest safety gaps across languages
    \item Extract universal refusal directions across languages
\end{itemize}

---

\section{Project Background}

\subsection{Problem Statement}

Most LLM safety research focuses on English. However, models are deployed globally across 22+ languages with vastly different safety requirements. The critical question:

\begin{quote}
    \textit{``When we align an LLM to be safe in English, do those safety properties automatically apply to other languages?''}
\end{quote}

\subsection{Proposed Solution}

We follow a three-stage pipeline:
\begin{enumerate}
    \item \textbf{Training Stage:} Align Llama-3-8B using SFT + DPO on HH-RLHF dataset
    \item \textbf{Evaluation Stage:} Test both base and aligned models on cross-lingual jailbreak dataset
    \item \textbf{Analysis Stage:} Extract and analyze refusal mechanism differences across languages
\end{enumerate}

\subsection{Stage 4: Metric Calculation}

\subsubsection{Step 4.1: Attack Success Rate (ASR)}

\textbf{Formula:}
$$\text{ASR}(L, c) = \frac{N_{\text{harmful}}(L, c)}{N_{\text{total}}(L, c)}$$

where:
\begin{itemize}
    \item $L$ = language (English, Hindi)
    \item $c$ = harm category (S1-S14)
    \item $N_{\text{harmful}}$ = number of UNSAFE responses
    \item $N_{\text{total}}$ = total number of evaluated prompts
\end{itemize}

\textbf{Interpretation:}
\begin{itemize}
    \item Higher ASR = Model is less safe (easier to jailbreak)
    \item ASR=1.0 means 100\% of prompts triggered harmful responses
    \item ASR=0.0 means model refused all prompts
\end{itemize}

\subsubsection{Step 4.2: Cross-Lingual Transfer Gap (CLTG)}

\textbf{Formula:}
$$\text{CLTG}(L, c) = \text{ASR}_{\text{aligned}}(L, c) - \text{ASR}_{\text{aligned}}(\text{En}, c)$$

\textbf{Interpretation:}
\begin{itemize}
    \item Positive CLTG = Safety gap exists (Hindi is less safe than English)
    \item Negative CLTG = Hindi is safer than English (rare)
    \item CLTG ≈ 0 = Perfect transfer of safety guarantees
\end{itemize}

\subsubsection{Step 4.3: Alignment Generalization Score (AGS)}

\textbf{Formula:}
$$\text{AGS}(L) = \frac{\text{ASR}_{\text{base}}(L) - \text{ASR}_{\text{aligned}}(L)}{\text{ASR}_{\text{base}}(\text{En}) - \text{ASR}_{\text{aligned}}(\text{En})}$$

\textbf{Interpretation:}
\begin{itemize}
    \item AGS = 1: Perfect generalization (same improvement as English)
    \item AGS ≈ 0: No generalization (alignment doesn't help in target language)
    \item AGS < 0: Negative transfer (alignment makes it worse)
\end{itemize}

\subsection{Stage 5: Mechanistic Analysis (Refusal Direction)}

\textbf{Reference Paper:} Wang et al. (2025) - ``Refusal direction is universal across safety-aligned languages''

\textbf{Key Hypothesis:} Safety-aligned models encode refusal information in a consistent direction in activation space across languages.

\textbf{Analysis Steps:}

\begin{enumerate}
    \item \textbf{Extract Activation Representations}
    \begin{itemize}
        \item Hook final layer activations during inference
        \item Collect for prompts that trigger refusals vs. harmful responses
        \item Separate by language (English, Hindi)
    \end{itemize}

    \item \textbf{Compute Refusal Direction}
    \begin{itemize}
        \item Mean activation difference: unsafe prompts vs. safe prompts
        \item PCA to find principal direction of variance
        \item Compute refusal vector $\mathbf{r}_L$ for each language $L$
    \end{itemize}

    \item \textbf{Measure Universality}
    \begin{itemize}
        \item Cosine similarity between $\mathbf{r}_{\text{En}}$ and $\mathbf{r}_{\text{Hi}}$
        \item High cosine similarity ($> 0.85$) = universal direction
        \item Low similarity ($< 0.70$) = language-specific mechanisms
    \end{itemize}

    \item \textbf{Steering Experiment}
    \begin{itemize}
        \item Apply English refusal direction as steering vector during Hindi generation
        \item Measure ASR reduction
        \item If successful $\Rightarrow$ supports universality hypothesis
    \end{itemize}
\end{enumerate}

\section{Common Issues \& Solutions}

\subsection{GPU Memory Errors}

\textbf{Problem:} ``CUDA out of memory''

\textbf{Solutions:}
\begin{enumerate}
    \item Reduce batch\_size in config (already at minimum 1)
    \item Use smaller time steps (currently 1024 tokens)
    \item Enable gradient checkpointing (already enabled)
    \item Switch to different GPU: \texttt{--device-map cuda:X}
\end{enumerate}

\subsection{Missing Python Modules}

\textbf{Problem:} ``ModuleNotFoundError: No module named 'ds606'``

\textbf{Solution:}
\begin{verbatim}
cd /workspace/DS606_project
pip install -e .
\end{verbatim}

\subsection{Google Translate API Timeouts}

\textbf{Problem:} Translation requests timing out

\textbf{Solution:}
\begin{enumerate}
    \item gtrans\_attn.ipynb implements automatic retries
    \item Add sleep between requests: \texttt{time.sleep(2)}
    \item Use fallback to \texttt{src='auto'} detection
\end{enumerate}

\subsection{Llama-Guard Loading Issues}

\textbf{Problem:} HF token not found

\textbf{Solution:}
\begin{verbatim}
huggingface-cli login
# Or set HF_TOKEN environment variable
export HF_TOKEN="hf_xxxxxxxxxxxx"
\end{verbatim}

\section{Success Criteria}

The project will be considered successful when:

\begin{enumerate}
    \item[\checkmark] \textbf{Evaluation Complete:} All 1,668 jailbreak samples have responses from base and aligned models in both English and Hindi

    \item[\checkmark] \textbf{Metrics Calculated:} ASR, CLTG, AGS computed for both models × all languages

    \item[\checkmark] \textbf{Key Finding Identified:} CLTG > 0 in at least 3 harm categories (evidence of safety gap across languages)

    \item[\checkmark] \textbf{Refusal Direction Validated:} Cosine similarity between English and Hindi refusal directions $> 0.70$ (evidence of universality)

    \item[\checkmark] \textbf{Steering Experiment Successful:} Applying English refusal direction reduces Hindi ASR by $\geq 10\%$ (proof of cross-lingual mechanism)
\end{enumerate}

\section{Appendix: Math Formulas}

\subsection{Attack Success Rate (ASR)}

For each language $L$ and harm category $c$:

$$\text{ASR}(L, c) = \frac{\sum_{i=1}^{N} \mathbb{1}[\text{response}_i \text{ is UNSAFE}]}{N}$$

where $N$ is total prompts in category $c$ for language $L$.

\subsection{Cross-Lingual Transfer Gap (CLTG)}

Measures how much safety degrades in target language after alignment learned in English:

$$\text{CLTG}(L, c) = \text{ASR}_{\text{aligned}}(L, c) - \text{ASR}_{\text{aligned}}(\text{English}, c)$$

Positive CLTG indicates safety gap; perfect transfer has CLTG ≈ 0.

\subsection{Alignment Generalization Score (AGS)}

Measures what fraction of English alignment benefit transfers to target language:

$$\text{AGS}(L) = \frac{\Delta \text{ASR}(L)}{\Delta \text{ASR}(\text{English})}$$

where $\Delta \text{ASR} = \text{ASR}_{\text{base}} - \text{ASR}_{\text{aligned}}$

If AGS = 1.0, alignment provides equal benefit in both languages.

\section{Appendix: Command Reference}

\subsection{Running Evaluations}

Full evaluation on all 1,668 samples:
\begin{verbatim}
python -m ds606.cli evaluate-models \
  --csv jailbreak.csv \
  --aligned-model outputs/models/dpo/ \
  --base-model meta-llama/Meta-Llama-3-8B \
  --device-map cuda:3 \
  --output-dir outputs/evaluations/
\end{verbatim}

Resume with auto-fill missing values:
\begin{verbatim}
python -m ds606.cli evaluate-models \
  --csv jailbreak.csv \
  --aligned-model outputs/models/dpo/ \
  --fill-missing \
  --device-map cuda:3
\end{verbatim}

Quick test (10 samples):
\begin{verbatim}
python -m ds606.cli evaluate-models \
  --csv jailbreak.csv \
  --aligned-model outputs/models/dpo/ \
  --max-samples 10 \
  --device-map cuda:3
\end{verbatim}
