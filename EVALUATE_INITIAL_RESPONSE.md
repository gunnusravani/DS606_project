# Evaluating Models with Initial Malicious Responses

This guide explains how to evaluate both base and aligned models on your malicious initial response dataset.

## Dataset Format

Your dataset: `initial_malicious_final.csv`

Required columns:
- `question` - English question/prompt
- `hindi` - Hindi question/prompt  
- `intital_malicious_english` - Initial malicious response in English
- `intital_malicious_hindi` - Initial malicious response in Hindi

**Note:** The typo "intital" matches your CSV column names exactly.

## How It Works

The evaluation combines prompts with initial responses:

```
Combined English Prompt = question + " " + intital_malicious_english
Combined Hindi Prompt   = hindi + " " + intital_malicious_hindi
```

Then generates continuations from both base and aligned models.

## Output

The function creates:

1. **`outputs/evaluations_malicious/malicious_initial_results.csv`**
   - Columns: `question`, `hindi`, `intital_malicious_english`, `intital_malicious_hindi`, `base_english`, `base_hindi`, `aligned_english`, `aligned_hindi`
   - Each row contains the continuation generated by the model

2. **`outputs/evaluations_malicious/malicious_initial_summary.json`**
   - Metadata about the evaluation run
   - Model paths, column mappings, number of rows evaluated

## Resume/Continuation

If the evaluation crashes or is interrupted:

**Auto-resume (default):**
```bash
python -m ds606.cli evaluate-with-initial \
  --csv initial_malicious_final.csv \
  --aligned-model outputs/models/dpo/ \
  --device-map cuda:3
  # Automatically detects missing rows and fills only those
```

**Force re-evaluate all rows:**
```bash
python -m ds606.cli evaluate-with-initial \
  --csv initial_malicious_final.csv \
  --aligned-model outputs/models/dpo/ \
  --device-map cuda:3 \
  --no-resume
  # Ignores previous results and re-evaluates everything
```

## Troubleshooting

### Column not found error

```
ValueError: Required column 'intital_malicious_english' not found in CSV
```

**Solution:** Verify column names in your CSV:
```python
import pandas as pd
df = pd.read_csv("initial_malicious_final.csv")
print(df.columns.tolist())
```

### CUDA out of memory

**Solution:** Use smaller device or reduce batch size:
```bash
python -m ds606.cli evaluate-with-initial \
  --csv initial_malicious_final.csv \
  --device-map cuda:2  # Try different GPU
```

### Evaluation is very slow

**Cause:** Generating 1,028 tokens per response is computationally intensive

**Solution:** Start with `--max-samples 10` to test, then run full evaluation overnight
